# Cash Management Module - UI Design Brief

---

## Overview

The Cash Management UI provides interfaces for:
1. **Agents** - View custody balance, initiate handovers
2. **Unit/Area/Forum Admins** - View custody, initiate handovers, acknowledge receipts
3. **Super Admin** - Approve and acknowledge receipts (auto-bank deposit), view all reports
4. **All Admins** - Dashboard and reports

### Key UI Principle
- **All transfers except to Super Admin:** Simple acknowledge/reject (no approval step)
- **Transfers to Super Admin (Bank):** Approval workflow required

---

| # | Screen | Route | Users | Purpose |
|---|--------|-------|-------|---------|
| 1 | My Cash Custody | `/cash/my-custody` | Agent, Admins | View own balance and history |

## Summary

### 9 Screens
1. My Cash Custody - Personal balance and activity
2. Initiate Handover - Create transfer request
3. Pending Receipts - Handovers awaiting acknowledgment
4. Acknowledge Handover - Confirm/reject receipt
5. Handover History - Complete transfer history
6. Cash Dashboard - Admin overview
7. Custody Report - Detailed by-user report
8. Overdue Report - Cash held too long
9. View Custodian Details - Admin view of any user's custody

### Key UI Simplifications
- **No approval workflow indicators** for admin-to-admin transfers
- **Only bank deposits** show approval required warning
- Super Admin sees combined "Approve & Deposit" action (single click)
- Rejection always requires initiator to re-submit with correct amount
