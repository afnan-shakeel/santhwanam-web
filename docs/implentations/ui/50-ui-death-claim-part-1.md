# Death Claims UI Design

---

## **Overview**

Death Claims module needs to handle:
1. Claim submission (by nominees/agents)
2. Claim verification & approval workflow
3. Contribution cycle creation & management
4. Member contributions collection
5. Benefit payout processing
6. Claim tracking & history

---

## **Recommended Structure: 4 Main Pages**

1. **Claims Dashboard** - Overview of all claims
2. **Claim Details** - Individual claim management
3. **Submit New Claim** - Claim submission form
4. **My Claims** - For nominees/members to track their claims

## **Summary**

**4 Main Pages:**
1. **Claims Dashboard** - Overview with filters, stats, action items
2. **Claim Details** - 4 tabs (Overview, Documents, Cycle, Timeline)
3. **Submit Claim** - 3-step wizard (Member, Nominee, Documents)
4. **My Claims** - Nominee tracking view

**Key Features:**
- Clear status indicators
- Progress tracking for contributions
- Document verification workflow
- Timeline/audit trail
- Mobile-responsive design
- Role-based views (Admin/Agent/Nominee)
