# Finance / General Ledger Module - Complete UI Specification

**Version:** 1.0  
**Last Updated:** January 2025  
**Status:** Draft

---

## Overview

The Finance/GL Module provides financial management capabilities for the cooperative society, including chart of accounts management, journal entry viewing, financial reports, and fiscal period controls. This module is primarily used by Super Admin and Finance Admin roles.

---

## Key Design Decisions

### 1. Read-Heavy Module
Most journal entries are **system-generated** from other modules (Membership, Wallets, Contributions, Claims). Manual journal entry creation is rare and restricted to authorized users.

### 2. Cash Custody Hierarchy
The Chart of Accounts includes a **parent-child structure** for cash accounts to track physical cash through the organizational hierarchy:
- 1000 Cash (Parent/Summary)
  - 1001 Cash - Agent Custody
  - 1002 Cash - Unit Admin Custody
  - 1003 Cash - Area Admin Custody
  - 1004 Cash - Forum Admin Custody

### 3. Calculated vs Stored Values
**Retained Earnings** is calculated (Revenue - Expenses) rather than stored.

### Validation Rules

- Account code: Required, 4 digits, unique
- Account name: Required, max 100 characters
- Account type: Required
- Parent account: Optional, but if selected, parent cannot itself have a parent (max 2 levels)
- Normal balance: Auto-determined (Asset/Expense = Debit, Liability/Revenue = Credit)

### Validation Rules

- Entry date: Required, not in closed period, not future
- Description: Required
- Minimum 2 lines
- Each line must have either debit OR credit (not both, not neither)
- Total debits must equal total credits
- All accounts must be active
- System accounts can be used (they're just protected from deletion)

# Part 6: Updates and Enhancement - (Accounts Receivable)

**Accounts Receivable** for tracking contribution obligations, the following work is required:

---

## 10.1 When Would A/R Be Needed?

A/R becomes relevant when:
1. **Contribution cycles create obligations** - When a death claim is created, all active members owe a contribution
2. **Cash collection is deferred** - Members without sufficient wallet balance must pay via agent cash collection
3. **Financial reporting needs** - Balance Sheet should reflect "money owed to organization"

---

## 10.2 New Chart of Accounts Required

| Code | Account Name | Type | Purpose |
|------|--------------|------|---------|
| 1200 | Contributions Receivable | Asset | Track uncollected contribution obligations |
| 4250 | Contribution Income (Unbilled) | Revenue (Contra) | Offset for unbilled revenue (optional) |

---

## 10.3 Accounting Treatment

### Option A: Recognize Revenue When Collected (Current Approach - Simpler)

No A/R needed. Revenue is recognized when cash is collected or wallet is debited.

```

  Dr: Member Wallet Liability    20.00
  Cr: Contribution Income        20.00

Collection via cash:
  Dr: Cash - Agent Custody       20.00
  Cr: Contribution Income        20.00
```

```

  Dr: Contributions Receivable   10,000.00

  Dr: Member Wallet Liability     6,000.00
  Cr: Contributions Receivable    6,000.00

  Dr: Cash - Agent Custody        1,000.00
  Cr: Contributions Receivable    1,000.00
```
